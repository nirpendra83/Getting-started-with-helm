üß± What is a Helm Chart?

  • A Helm chart is a pre-packaged collection of Kubernetes resource definitions used to deploy applications or services.
  • It contains manifest templates (e.g., Deployments, Services, ConfigMaps) and a values.yaml file to customize the deployment.
  • Charts enable repeatable, consistent, and version-controlled application deployment in Kubernetes.

üì¶ What is a Helm Repository?

  • A Helm repository is a storage location for one or more Helm charts.
  • Repositories can be public (like Bitnami or ArtifactHub) or private/internal.
  • You must add a repository to your Helm client before you can search or install charts from it.

üîß Common Helm CLI Commands

  • Add a repository:

    helm repo add <name> <url>

  • Update repository cache:

    helm repo update

  • Search for charts:

    helm search repo <keyword>

  • Install a chart:

    helm install <release-name> <chart-name>

  • List all releases:

    helm list

  • Upgrade a release:

    helm upgrade <release-name> <chart-name>

  • Rollback to previous revision:

    helm rollback <release-name> <revision-number>

  • Uninstall a release:

    helm uninstall <release-name>

üìò Additional Helm Concepts

  • values.yaml: Defines default configuration for a chart and can be overridden using --values or --set.
  • Release: A deployed instance of a Helm chart inside your Kubernetes cluster.
  • Templates: Kubernetes YAML manifests parameterized using Go templating syntax.
  • Dependencies: Charts can depend on other charts, specified in the Chart.yaml and stored under charts/.

Explanation

  • Chart.yaml: Defines chart metadata such as name, version, and maintainers.
  • values.yaml: Contains default values that can be overridden during installation or upgrade.
  • charts/: Stores dependent charts (subcharts).
  • templates/: Contains Go-templated Kubernetes manifests rendered into actual YAML.
  • README.md: Optional file that documents the chart’s purpose, usage, and examples.

This modular structure allows Helm to manage complex Kubernetes applications efficiently, supporting reuse, versioning, and customization.
